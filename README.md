# Практическая работа с объектами и классами.

Описание проекта
Этот проект представляет собой ядро интернет-магазина, реализованное на Python с использованием классов. 
В нем реализованы классы Product, Category, а также дочерние классы для различных типов товаров, а также модули 
для работы с данными, выводом и тестированием.

1 -я функция Product:
имеет следующие свойства:
 название (name) описание (description), цена (price), количество в наличии (quantity).
Добавлено в функционал:
Принимаем цену из привата

Возвращает строку с ценой или сообщением об ошибке

Изменение цены в случае ее понижения с согласия пользователя

Добавлены дочерние классы принимающие родительский класс Product
для работы с категорией Smartphone и Lawngrass.
 
Добавлена защита от суммирования разных классов продуктов в одну сумму.



2-я функция Category:
имеет следующие свойства:
название (name), описание (description), список товаров категории (products)

Добавляем новый продукт

Возвращает форматированную строку с информацией о продуктах

Добавляет новые продукты в категорию и обновляет счётчик

Добавлена защита от суммирования разных классов в одну сумму.



** Модуль utils
1-я функция: Чтение файла data/products.json и возврат запрашиваемых свойств продукта.
 модуль BaseProduct
Реализует абстракт метод создания универсального каркаса на котором строятся родительские классы.
 модуль Print_mixin
Реализация универсального модуля вывода на печать.



## Тестирование
### Для тестирования вам потребуется:
установить интерпретатор Pycharm

установить фреймворк pytest  командой poetry add --group dev pytest

Настроить pytest из меню по пути:

current file -> edit configurations -> '+' -> add new configuration -> pytest -> 
script path -> указать директорию тестов и директорию проекта в целом.

Тесты проекта находятся в корне проекта в директории tests.
Conftest содержит фикстуры для тестирования.
Test_category тестирование категорий товаров
Test_product тестирование свойств продукта - сверка вывода ожидаемого результата.
Test_smartphone проверка работоспособности модуля и исключение ошибок.
Test_lawngrass проверка работоспособности модуля и исключение ошибок.
Test_print_mixin тестирование модуля вывода на печать с разных видов классов с реализацией условия наличия аргументов.

Установка и запуск
1. Клонирование репозитория
bash
git clone https://github.com/burilinas-hub/ObjectOrientProgramming.git

3. Установка виртуального окружения и зависимостей
Рекомендуется использовать Poetry для управления зависимостями.

Установка Poetry (если еще не установлен)
bash
pip install poetry

Установка зависимостей проекта
bash
poetry install
Это установит все необходимые пакеты, указанные в pyproject.toml.

Настройка и запуск тестов
3. Активировать виртуальное окружение
bash
poetry shell

4. Установить тестовые зависимости
bash
poetry add --group dev pytest pytest-cov

6. Настройка pytest
Настройте pytest, следуя инструкции:

В меню Run -> Edit Configurations
Добавьте новую конфигурацию pytest
Укажите путь к тестам (например, tests)
Укажите корень проекта
6. Запуск тестов
bash
pytest

7. Проверка покрытия кода
Для проверки покрытия тестами выполните:

bash
pytest --cov=src --cov-report=html

После выполнения команды откроется файл: htmlcov/index.html, где вы сможете увидеть процент покрытия тестами. 
Рекомендуется иметь покрытие выше 75% для более надежной проверки.
